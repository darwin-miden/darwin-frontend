3. ARCHITECTURE — AUJOURD'HUI vs CIBLE
──────────────────────────────────────────────
Aujourd'hui, 3 rails qui convergent sur le même slot-10 :
  1. ETH wallet  → Epoch → relay custodial → atomic_deposit_note
  2. Miden wallet → atomic deposit note direct
  3. Self-custody → browser écrit la position sur un controller
     NoAuth (workaround : n'importe qui peut soumettre un tx bundle)

Cible avec network accounts :
  Le controller devient un network account. Les deposits sont des
  notes taguées vers lui ; le Network Transaction Builder (NTX) du
  node les consume et exécute le MASM du controller — qui valide
  tout (montants, NAV, mint). Le relay disparaît, le NoAuth
  disparaît. Zéro opérateur, zéro backend, validation dans le code.

Levier : "Your NTX infra is already deployed on testnet — the
binary, GetNetworkNoteStatus RPC, everything. Our controller MASM
is ready, our deposit notes are already the right shape. We could
be your first real DeFi protocol on network transactions."
